# v0.1.3

## Features

- **Exponential-then-linear adaptive strategy:** Parallelization factor now starts at 2 (instead of CPU count) and doubles each evaluation window while throughput improves. Once throughput degrades, it reverts to half and switches to linear +1/-1 probing. CPU count is still reported per worker but no longer drives the initial factor.

- **Adaptive timeout response:** The parallelization factor is halved immediately when a worker reports a task timeout, rather than waiting for the next evaluation window. This also exits the doubling phase so the controller doesn't aggressively scale back up.

## Fixes

- **Docker build with hatch-vcs:** Fixed `uv sync` failing inside Docker due to missing `README.md` and absent git history. The Dockerfile now copies `README.md` and sets `SETUPTOOLS_SCM_PRETEND_VERSION` as a fallback.
